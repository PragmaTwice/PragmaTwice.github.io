C++ 标准不需要动态（运行期）反射

或者说，我认为一旦有了静态反射，动态反射就完全可以由社区提供，比如 RTTR，当然我更倾向于认为大部分需求都能用静态反射解决。

所有说可以用 C++ 宏实现反射的人都是不负责任，宏的部分功能应该被静态反射淘汰，这种极其落后的语言设施应该被消灭掉。
